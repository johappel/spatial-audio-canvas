# Spatial Audio Canvas

Ein browserbasierter Klangraum für gemeinsames Hören, Sprechen und Anwesendsein – ohne Kamera, ohne Account, ohne 3D-Steuerung.

## Was ist der Spatial Audio Canvas?

Der Spatial Audio Canvas ist eine einfache Webanwendung, mit der Menschen gemeinsam einen digitalen Klangraum betreten können. 

Der Spatial Audio Canvas ermöglicht Begegnung durch hörbare Sitzordnung: Menschen nehmen an einfachen Gesprächsinseln Platz, hören Stimmen links, rechts, nah oder weiter entfernt und sehen an pulsierenden Lichtpunkten, wer spricht. Die Technik bleibt so unsichtbar wie möglich.

Anders als klassische Videokonferenzen oder 3D-Welten setzt dieses Konzept nicht auf Kamera, Avatare oder freie Bewegung.

Stattdessen sehen Nutzer:innen eine ruhige, anklickbare Karte mit wenigen Orten. Diese Orte können zum Beispiel heißen:

- Meer
- Kapelle
- Gespräch
- Konzert
- Podcast
- Stille
- Erzählfenster

Mit einem Klick wählt man einen Ort aus. Der Klang verändert sich entsprechend: Stimmen, Musik, Podcasts oder Atmosphäre wirken näher oder ferner.

Das Ziel ist ein niedrigschwelliger digitaler Begegnungsraum, der besonders für ältere Menschen, inklusive Gruppen, Bildungsarbeit und gemeinsames Hören geeignet ist.

## Warum kein klassischer 3D-Raum?

Viele virtuelle Räume setzen voraus, dass Nutzer:innen eine Figur durch eine Welt steuern. Dafür braucht man oft Maus, Tastatur, WASD-Steuerung oder Erfahrung mit Computerspielen.

Für viele Zielgruppen ist das eine Hürde.

Der Spatial Audio Canvas ersetzt diese Logik:

| Klassischer 3D-Raum | Spatial Audio Canvas |
|---|---|
| Avatar steuern | Ort anklicken |
| WASD / Mausblick | große Buttons |
| visuelle Welt im Mittelpunkt | Klang im Mittelpunkt |
| Identifikation über Figur | Präsenz über Lichtpunkt und Stimme |
| oft komplexe Bedienung | reduzierte, ruhige Oberfläche |

Der Raum ist hier vor allem ein akustischer Raum.

## Zentrale Funktionen

### 1. Zugang per Link

Teilnehmende öffnen einen Link, geben einen Namen oder ein Pseudonym ein und betreten den Raum.

Keine Registrierung. Kein Profil. Keine Installation.

### 2. Keine Kamera

Die Anwendung ist bewusst audiozentriert. Es gibt keine Videokonferenz und keine Kameraansicht.

Das senkt technische, soziale und datenschutzbezogene Hürden.

### 3. Klangorte

Der Raum besteht aus mehreren Klangorten. Jeder Ort kann eigene Audioquellen, Gespräche oder Atmosphären besitzen.

Beispiele:

- Am Meer hört man Wellen.
- In der Kapelle hört man eine Orgel oder eine Lesung.
- Im Gesprächskreis hört man andere Teilnehmende deutlich.
- Beim Konzert hört man Musik näher.
- In der Stille-Ecke werden Stimmen und Musik gedämpft.

### 4. Räumliches Hören

Stimmen und Audioquellen werden je nach Aufenthaltsort lauter, leiser oder räumlich verschoben.

Die Nutzer:innen erleben dadurch Nähe und Distanz, ohne sich durch eine 3D-Welt bewegen zu müssen.

### 5. Sichtbare Sprechaktivität

Teilnehmende erscheinen als einfache Lichtpunkte oder Namenspunkte. Wenn jemand spricht, pulsiert der Punkt.

So ist erkennbar, wer gerade redet, ohne dass eine Kamera nötig ist.

### 6. Barrierearme Bedienung

Die Oberfläche ist auf einfache Bedienbarkeit ausgelegt:

- große Schaltflächen
- klare Beschriftungen
- Tastaturbedienung
- Screenreader-Unterstützung
- hoher Kontrast
- einfache Sprache
- keine Präzisionsbewegungen mit der Maus
- keine Spielsteuerung

## Anwendungsmöglichkeiten im Bildungskontext

Der Spatial Audio Canvas eignet sich besonders für Lernformen, bei denen Hören, Wahrnehmen, Gespräch, Atmosphäre und Reflexion wichtig sind.

## 1. Religionsunterricht und spirituelle Bildung

### Digitale Resonanzräume

Schüler:innen oder Erwachsene betreten eine Abfolge von Klangorten:

- Ankommen
- Hören
- Klage
- Dank
- Stille
- Antwort

Jeder Ort kann eine eigene Atmosphäre und einen eigenen Impuls enthalten.

Beispiel:

- Im Raum „Hören“ erklingt ein kurzer Text.
- Im Raum „Klage“ hört man gedämpfte, schwere Klänge.
- Im Raum „Antwort“ können Teilnehmende leise miteinander sprechen.

Der Raum ermöglicht eine Form digitaler Liturgie oder spiritueller Erkundung, ohne dass eine Videokonferenzsituation entsteht.

### Bibeltexte räumlich erfahren

Ein biblischer Text wird nicht nur gelesen, sondern räumlich gehört.

Beispiel „Sturmstillung“:

- Ort 1: See und Wind
- Ort 2: Boot
- Ort 3: Angst der Jünger
- Ort 4: Stille nach dem Sturm

Lernende bewegen sich nicht durch Animationen, sondern durch Klangperspektiven.

### Ethische Dilemmata als Hörstationen

Mehrere Positionen zu einer ethischen Frage werden als Orte angelegt.

Beispiel KI und Verantwortung:

- Ort „Entlastung“
- Ort „Kontrolle“
- Ort „Gerechtigkeit“
- Ort „Menschenwürde“

Schüler:innen hören Impulse und wechseln anschließend in Gesprächsorte.

## 2. Sprachbildung und Fremdsprachenunterricht

### Hörverstehen in Stationen

Statt alle hören denselben Text frontal, gibt es verschiedene Hörstationen:

- Alltagssituation
- Interview
- Dialog
- Nachricht
- Musik oder Geräuschkulisse

Lernende wählen eine Station, hören mehrfach und tauschen sich anschließend in Kleingruppen aus.

### Aussprache und Sprechmut

Da keine Kamera aktiv ist, kann die Hemmschwelle sinken. Lernende sprechen über Audio, sehen aber nur reduzierte Punkte.

Das kann besonders für zurückhaltende Schüler:innen hilfreich sein.

### Rollenbasierte Dialogräume

Ein Raum kann verschiedene Rollen enthalten:

- Bahnhof
- Café
- Hotelrezeption
- Arztpraxis
- Marktstand

Lernende wechseln den Ort und üben passende Dialoge.

## 3. Musikunterricht und kulturelle Bildung

### Musikstücke räumlich erkunden

Ein Musikstück kann in verschiedene Hörperspektiven aufgeteilt werden:

- Rhythmus
- Melodie
- Bass
- Stimme
- Instrumentengruppe

Die Lernenden wechseln zwischen Klangorten und beschreiben, was sie wahrnehmen.

### Konzertpädagogik

Vor oder nach einem Konzert kann ein digitaler Raum entstehen:

- Einführung
- Instrumente
- Komponist:in
- Hörbeispiele
- Gespräch

Der Spatial Audio Canvas wird zum Vorraum kultureller Erfahrung.

### Klanglandschaften gestalten

Lernende erstellen eigene Klangorte und diskutieren, wie Klang Bedeutung erzeugt.

Beispiele:

- Stadt
- Wald
- Meer
- Krieg
- Frieden
- Erinnerung

## 4. Geschichte und Erinnerungskultur

### Historische Hörstationen

Eine historische Epoche kann als Klangkarte dargestellt werden.

Beispiel Industrialisierung:

- Fabrik
- Arbeiterwohnung
- Bahnhof
- Parlament
- Kirche

An jedem Ort hören Lernende kurze Quellen, Geräusche oder Perspektiven.

### Multiperspektivität

Verschiedene Stimmen werden räumlich getrennt:

- Zeitzeug:in
- Politiker:in
- Arbeiter:in
- Kind
- Kritiker:in

Die räumliche Struktur macht deutlich: Geschichte besteht aus Perspektiven.

### Erinnerungsräume

Für Gedenktage können ruhige digitale Räume gestaltet werden:

- Stimmen hören
- Namen wahrnehmen
- Stille halten
- Gespräch ermöglichen

Gerade hier ist der Verzicht auf Kamera und Avatar ein Vorteil, weil die Form würdevoller und weniger spielerisch wirkt.

## 5. Deutschunterricht und Literatur

### Literarische Räume

Texte können als Klangräume erschlossen werden.

Beispiel Ballade:

- Ort der Handlung
- Stimme des Erzählers
- innere Stimme der Figur
- Konfliktort
- Schlussbild

Die Lernenden hören, beschreiben und interpretieren.

### Szenisches Lesen ohne Bühne

Rollen werden akustisch verteilt. Wer spricht, wird sichtbar hervorgehoben. Die Klasse erlebt ein Hörspiel ohne Kamera.

### Schreibanlässe

Nach dem Besuch eines Klangraums schreiben Lernende:

- einen inneren Monolog
- eine Raumbeschreibung
- eine Reflexion
- ein Gedicht
- eine Fortsetzung

## 6. Inklusive Bildung

Der Spatial Audio Canvas kann besonders dort hilfreich sein, wo visuelle Präsenz, Kamera oder komplexe Steuerung ausschließend wirken.

Potenziale:

- Teilnahme ohne sichtbares Bild
- geringe soziale Exposition
- einfache Interaktion
- Möglichkeit zum Zuhören ohne Sprechpflicht
- klare akustische und visuelle Rückmeldungen
- nutzbar in Kleingruppen

Wichtig ist jedoch: Barrierefreiheit muss konkret getestet werden. Der Canvas ersetzt keine professionelle Assistenztechnologie, kann aber niedrigschwellige Teilhabe unterstützen.

## 7. Senior:innenbildung und intergenerationelles Lernen

Der ursprüngliche Impuls des Projekts ist ein digitales Hörfenster für Menschen, die nicht einfach an öffentlichen Orten teilnehmen können.

Mögliche Szenarien:

- gemeinsames Podcast-Hören
- Erzählcafé
- Musik am Nachmittag
- Andacht
- digitales Vorlesen
- Austausch zwischen Schule und Senior:innenheim
- biografisches Erzählen
- lokale Geschichte hören

Beispiel:

Eine Schulklasse gestaltet einen Klangraum „Unser Stadtteil früher und heute“. Senior:innen betreten den Raum, hören Aufnahmen und erzählen eigene Erinnerungen. Schüler:innen hören zu und stellen Fragen.

## Pädagogische Leitfragen

Beim Einsatz im Unterricht sollten Lehrkräfte nicht nur fragen: „Funktioniert die Technik?“, sondern:

- Was verändert sich, wenn Lernen über Hören organisiert wird?
- Wer kommt eher zu Wort, wenn keine Kamera aktiv ist?
- Wie entsteht Nähe ohne Sichtbarkeit?
- Wie unterstützt Klang die Wahrnehmung von Perspektiven?
- Wann hilft räumliche Distanz beim Gespräch?
- Wann braucht es Moderation?
- Wie werden stille Teilnehmende einbezogen?

## Moderationsformen

### Freier Raum

Alle bewegen sich selbständig zwischen Orten.

Geeignet für:

- offene Erkundung
- Pausenräume
- informelle Begegnung

### Geführter Raum

Eine Moderation sagt, welcher Ort als nächstes besucht wird.

Geeignet für:

- Unterricht
- Andachten
- Workshops
- Einführungen

### Hörkreis

Alle starten an einem Ort und hören gemeinsam. Danach wechseln sie in Gesprächsbereiche.

Geeignet für:

- Podcastarbeit
- Literatur
- Musik
- Quellenarbeit

### Stationenlernen

Kleingruppen besuchen verschiedene Orte und bearbeiten Aufgaben.

Geeignet für:

- Schule
- Fortbildung
- Projektarbeit

## Datenschutz und ethische Grundsätze

Der Spatial Audio Canvas soll datensparsam gestaltet sein.

Grundsätze:

- keine Kamera
- keine Registrierung
- keine Aufzeichnung
- keine dauerhafte Speicherung von Namen
- keine Analyse- oder Werbetracker
- transparente Hinweise beim Betreten
- einfache Möglichkeit, Mikrofon stummzuschalten
- Räume nur über bewusste Linkweitergabe zugänglich

Für Bildungseinrichtungen sollte vor einem produktiven Einsatz geprüft werden:

- Wo wird die Anwendung gehostet?
- Welche Signalling- oder Relay-Dienste werden genutzt?
- Werden IP-Adressen verarbeitet?
- Gibt es Auftragsverarbeitungsverträge, falls notwendig?
- Sind Audiodaten flüchtig?
- Ist der Einsatz mit den lokalen Datenschutzvorgaben vereinbar?

## Technische Grundidee

Die Anwendung nutzt moderne Browsertechnologien:

- Web Audio API für räumliche Klangverarbeitung
- WebRTC für Audioverbindungen zwischen Teilnehmenden
- ein einfaches Signalling-System für Raumbeitritt und Verbindungsaufbau
- JSON-Dateien zur Definition von Räumen und Klangorten

Für einen ersten Prototypen soll kein eigener Medienserver nötig sein. Kleine Gruppen können über WebRTC-Mesh getestet werden. Für größere Gruppen kann später ein anderer Kommunikationsansatz ergänzt werden.

## Beispielraum: Hörfenster am Meer

Orte:

1. **Meer**  
   Leises Wellenrauschen. Gut zum Ankommen.

2. **Erzählfenster**  
   Hier läuft ein Podcast oder eine vorgelesene Geschichte.

3. **Gesprächskreis**  
   Hier hören sich Teilnehmende klar und direkt.

4. **Stille Ecke**  
   Stimmen und Quellen sind gedämpft. Geeignet zum Pausieren.

5. **Musik**  
   Eine Musikquelle ist räumlich gebunden.

## Beispielraum: Unterrichtsimpuls

Thema: „Wie klingt Verantwortung?“

Orte:

- Entscheidung
- Zweifel
- Folgen
- Stimmen anderer
- Reflexion

Ablauf:

1. Lehrkraft begrüßt die Gruppe.
2. Lernende hören kurze Impulse an verschiedenen Orten.
3. Sie sammeln Eindrücke.
4. Kleingruppen sprechen an Gesprächsorten.
5. Abschluss im gemeinsamen Reflexionsort.

## Projektvision

Der Spatial Audio Canvas soll kein weiteres Meetingtool werden.

Er soll ein einfaches digitales Fenster öffnen:

- für Menschen, die nicht mobil sind
- für Gruppen, die gemeinsam hören möchten
- für Lernende, die Perspektiven akustisch erkunden
- für spirituelle und kulturelle Bildung
- für Unterricht, der Wahrnehmung ernst nimmt
- für Begegnung ohne Kamerazwang

Die wichtigste Erfahrung lautet:

**Ich bin nicht allein. Ich höre andere. Ich kann näher kommen. Ich kann Abstand nehmen. Ich bin anwesend.**

## Projektstruktur und Entwicklung

Dieses Repository enthaelt ein lauffaehiges, erweiterbares Fundament (Vite + TypeScript + Lit).

### Schnellstart

```bash
npm install
npm run dev        # Frontend auf http://localhost:5173
npm run signaling  # optional: WebSocket-Signalling auf ws://localhost:8766
npm test           # Unit-Tests
npm run build      # Produktionsbuild nach dist/
```

### Architektur und Erweiterbarkeit

- **Kernel** (`src/core/`): EventBus, nanostores-Store, PluginHost + Manifest, UI-Regionen, ServiceRegistry.
- **Plugins** (`src/plugins/`): Chat, Emotes, Klanggesten, Spiele - registrieren sich ueber ein Manifest und haengen ihre UI in definierte Regionen.
- **Audio** (`src/audio/`): AudioEngine plus erweiterbare **Klangnodes** (`KlangNodeRegistry`), Spatializer, Pegelanalyse, Ambient-Player.
- **Medien** (`src/media/`): `MediaSourceRegistry` fuer neue Medientypen.
- **RTC** (`src/rtc/`): austauschbares Signalling, WebRTC-Mesh, DataChannel-Bus, Presence.
- **Raum** (`src/room/`): Sitzgeometrie und egozentrische Perspektive.

Neue Interaktionen (weitere Emotes, Spiele, Klanggesten) werden als Plugin ergaenzt, neue Klang-Typen ueber die KlangNode-Registry, neue Medien ueber die MediaSource-Registry - ohne den Kern zu aendern.

### Deployment (GitHub Pages + Signalling)

Das Frontend laeuft vollstaendig statisch auf GitHub Pages (Workflow: `.github/workflows/deploy-pages.yml`). Die lokale Audio-Demo funktioniert dort ohne Server. Fuer echte Mehrpersonen-Gespraeche wird ein kleiner Signalling-Dienst benoetigt - siehe [docs/install-ws-service.md](docs/install-ws-service.md).

### Realitaetscheck

- GitHub Pages: Frontend + lokale Demo, kein Server noetig.
- Echte Stimmen: Signalling (extern), STUN (oeffentlich), TURN (optional).
- Audiostreams laufen Peer-to-Peer; ueber den Signalling-Server laufen keine Audiodaten.

### Hinweis zu Dependencies

Die gemeldeten `npm audit`-Warnungen betreffen ausschliesslich die Dev-Toolchain (esbuild-Dev-Server via Vitest) und nicht das ausgelieferte statische Build-Artefakt.
